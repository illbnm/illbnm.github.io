Integer包装类型



Integer 在底层的缓存是[-128,127] 缓存 (空间换时间) ,相当于常量池,而超过这个数字范围,就是在堆上开辟的空间存储的


a和b中的100在Integer的缓存中,因而调用的是同一块地址

因为c和d都在堆上开辟的,因而地址不相同

String

  • <=1.6 在方法区 >=1.7在堆区
  • String在底层是由final修饰的.因此一个字符串对象创建后它的值不能改变,改变
    String对象实际上在底层将String转换为它的包装类型StringBuilder
  • 创建String对象

虽然两种方式都是返回了一个String对象的引用,对于第一种,jvm会马上在堆上创建一个String对象,然后将该对象的引用返回
第二种 jvm首先会在内部维护的strings pool中通过String的 equels 方法查找是对象池中是否存放有该String对象，如果有，则返回已有的String对象给用户,而不会在heap中重新创建一个新的String对象；如果对象池中没有该String对象，jvm则在heap中创建新的String对象，将其引用返回给用户，同时将该引用添加至strings pool

intern() 功能

  • 如果常量池中存在当前字符串, 就会直接返回当前字符串. 如果常量池中没有此字符串, 会将此字符串放入常量池中后, 再返回
    主要目的就是将常量放到常量池并返回
